## 功能概述

Chrome 扩展，一键提取网页正文，保存为 Markdown 文件。

**核心能力：**
- 使用 Mozilla Readability 算法提取文章正文
- 使用 Turndown 将 HTML 转为 Markdown
- 保存为带 YAML frontmatter（标题、来源、日期）的 `.md` 文件
- 保存成功/失败均记录日志，支持导出
- 可自定义保存路径（绝对路径）

## 使用方法

### 安装

1. 打开 Chrome → `chrome://extensions/`
2. 右上角开启「开发者模式」
3. 点击「加载已解压的扩展程序」
4. 选择 `~/桌面/肘子文章提取器/`

### 保存文章

点击工具栏的 🐷 **肘子** 图标 → 点击 **「保存当前页面」**。

### 保存路径设置

默认保存路径：`/home/bao/下载/`（下载目录根）

可在设置页面修改为任意下载目录内的绝对路径，例如：
- `/home/bao/下载/文章` → 保存到 `下载/文章/标题.md`
- `/home/bao/下载/技术/前端` → 保存到 `下载/技术/前端/标题.md`
- 留空 → 直接保存到 `下载/标题.md`

> ⚠️ 仅支持 `下载/` 目录内的路径。超出下载目录的路径会自动回退到下载目录根。

### 日志

- 所有保存操作自动记录在扩展的本地存储中
- 在设置页面的「保存日志」标签页可查看和导出
- 导出文件名为 `肘子日志_YYYY-MM-DD.md`，保存到下载目录

## 文件结构

```
肘子文章提取器/
├── manifest.json          # Chrome 扩展配置（Manifest V3）
├── content.js             # 正文提取 + 转 Markdown（注入到页面中运行）
├── background.js          # 后台服务工作者（处理下载、通知、日志导出）
├── popup.html + popup.js  # 点击工具栏图标弹出的界面
├── options.html + popup.js# 设置页面（路径配置、日志查看）
├── lib/
│   ├── Readability.js     # Mozilla 正文提取算法（~90KB）
│   └── turndown.js        # HTML → Markdown 转换库（~27KB）
├── icon.png               # 扩展图标
├── README.md              # 简介
├── ERRORS.md              # 错误码参考文档
└── 功能说明.md             # 本文件
```

### 核心文件说明

| 文件 | 作用 |
|------|------|
| `manifest.json` | 扩展声明：权限（activeTab, notifications, downloads, storage, scripting）、主入口、content script 注入规则 |
| `content.js` | 注入到每个 http/https 页面，监听 `extract` 消息，用 Readability + Turndown 提取并返回 Markdown |
| `background.js` | Service Worker：弹窗通知、导出日志、处理下载请求 |
| `popup.js` | 点击图标的弹窗逻辑：获取当前标签页、调 content script 提取、触发下载 |
| `options.js` | 设置页面逻辑：路径保存/读取、日志列表渲染/导出/清空 |

## 错误码参考

| 错误码 | 含义 | 常见原因 |
|--------|------|---------|
| `NO_CONTENT` | 无可提取正文 | 不是文章页，或页面加载不全 |
| `NOT_AN_ARTICLE` | 不是文章类型 | 视频页、图片页、首页等 |
| `PAGE_TOO_SHORT` | 正文过短 | 页面加载不全或只有几句话 |
| `BLOCKED_BY_LOGIN` | 需要登录/付费 | 付费墙、会员专属内容 |
| `EMPTY_PAGE` | 页面内容为空 | 网站崩溃、网络失败、SPA 未加载 |
| `CONTENT_SCRIPT_NOT_READY` | 内容脚本未加载 | 刷新页面即可 |
| `DOWNLOAD_FAILED` | 下载失败 | 磁盘空间不足、路径不能创建 |
| `UNSUPPORTED_URL` | 不支持的链接 | chrome://、file:// 等内部页面 |
| `TIMEOUT` | 操作超时 | 网络慢、页面太大 |
| `UNKNOWN` | 未知错误 | 查看控制台日志 |

详细说明见 `ERRORS.md`。

## 常见问题

### 保存后文件在哪？

默认在 `~/下载/标题.md`。可在扩展设置页面修改保存路径。

### 保存时提示"刷新页面后再试"

内容脚本未注入，刷新当前页面即可。通常发生在刚打开的标签页或浏览器内部页面。

### 保存路径修改后不生效？

Chrome 扩展的 storage 是按扩展 ID 隔离的。如果重新加载扩展后 ID 变了，需要重新设置路径。路径存储在 `chrome.storage.sync`。

### 日志存在哪里？

存在浏览器本地存储（`chrome.storage.local`）中，不是文件系统。可在设置页面的「保存日志」标签页查看或导出。
